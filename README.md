## 3.1. Тема и концепция

### Название приложения
**PeriHub** — это веб-приложение для выбора и покупки периферийных устройств для ПК: клавиатур, мышей, наушников и других аксессуаров

### Краткое описание
PeriHub помогает геймерам, офисным сотрудникам и IT-специалистам находить идеальные устройства для работы и игр. В приложении можно сравнить характеристики разных моделей, изучить отзывы, отслеживать цены и покупать периферию. 

### Целевая аудитория
| Группа | Потребности |
|--------|-------------|
| Геймеры | Игровые мыши с высоким DPI, механические клавиатуры с подсветкой, удобные гарнитуры|
|Офисные сотрудники | Эргономичные мыши, тихие клавиатуры, качественные гарнитуры для видеозвонков|
|IT-специалисты | Надежные устройства для работы, механические клавиатуры для программирования|

### Ключевой функционал
1. Регистрация и личный кабинет с историей заказов
2. Каталог периферии с фильтрацией (тип, бренд, цена, характеристики)
3. Детальная страница товара с фото, описанием и отзывами
4. Корзина покупок и оформление заказа
5. Система сравнения товаров
6. Избранное для откладывания понравившихся товаров
7. Поиск по названию и категориям
8. Админ-панель для управления товарами и заказами

## 3.3. Технологический стек

### Frontend

| Технология | Обоснование выбора |
|------------|--------------------|
| **HTML5** | Базовый стандарт разметки, поддерживается всеми браузерами |
| **CSS3** | Стилизация интерфейса и адаптивный дизайн для мобильных устройст|
| **Vue.js 3 (Composition API)** | Vue обеспечивает отличную производительность и простоту разработки. Composition API позволяет гибко организовывать логику компонентов. Для интернет-магазина это идеальный выбор благодаря удобной работе со списками (директива v-for для отрисовки карточек товаров) и условным отображением (v-if/v-else для состояний загрузки и ошибок). Vue имеет более пологую кривую обучения по сравнению с React, что ускорит разработку|
| **Vue Router** | Официальный роутер для Vue с глубокой интеграцией во фреймворк. Позволяет легко организовать навигацию по разделам магазина, защитить приватные маршруты (корзина, личный кабинет) с помощью навигационных guards и работать с динамическими параметрами (/product/:id для страниц товаров)|
| **Pinia** | Официальный state-менеджмент для Vue, который пришел на смену Vuex. Идеально подходит для управления состоянием корзины покупок, избранного и фильтров каталога. Имеет отличную поддержку TypeScript, минимальный бойлерплейт и простой API. В отличие от Redux, Pinia интуитивно понятна и легко интегрируется с Vue.|
| **Axios** | HTTP-запросы к API: автоматический JSON, удобная обработка ошибок, отмена запросов при поиске (важно для автодополнения)|

### Backend

| Технология | Обоснование выбора |
|------------|--------------------|
| **Node.js** | Единый язык для фронтенда и бэкенда (JavaScript), огромное сообщество, множество готовых модулей через npm|
| **Express.js** | Минималистичный фреймворк для создания REST API, минимум кода, легкость в освоении|
| **PostgreSQL** | Промышленный стандарт, поддержка транзакций (важно для заказов), надежность, облачный хостинг Neon |


### Инфраструктура

| Компонент | Выбор | Обоснование выбора |
|-----------|---------|-------------|
| Хостинг фронтенда | **GitHub Pages** | Бесплатно, интегрировано с репозиторием |
| Хостинг бэкенда | **Render** | Бесплатный тариф для Node.js, авто-деплой из GitHub |
| База данных | **Neon PostgreSQL** | Бесплатный облачный хостинг PostgreSQL, автоматические бэкапы, удобный интерфейс |

## 3.4. Визуальное проектирование интерфейса

**Выбран вариант: В — Подробное текстовое описание с разметкой**

### Страница: Главная (/)
**Тип доступа:** публичный  
Основные блокы:
1.Шапка: логотип «PeriHub», меню (Главная, Каталог, Сравнение, Войти), иконка корзины с счетчиком
2.Герой-секция: заголовок «Периферия для побед», подзаголовок, кнопка «Перейти в каталог»
3.Секция «Категории»: 4 карточки (Клавиатуры, Мыши, Наушники, Аксессуары) — ссылки на /catalog?type=...
4.Секция «Популярные товары»: 4 карточки товаров (фото, название, цена, рейтинг) — данные из GET /api/products?popular=true
5.Секция «Новинки»: 4 карточки новых поступлений — данные из GET /api/products?new=true
6.Футер: контакты, ссылки на соцсети, информация о доставке
Динамический контент: блоки 4 и 5 загружаются через API

---

### Страница: Аутентификация (/auth/login, /auth/register)
Тип доступа: публичный
Основные блоки (для регистрации):
1.Форма: поля «Логин», «Email», «Пароль», «Подтверждение пароля», кнопка «Зарегистрироваться»
2.Ссылка «Уже есть аккаунт? Войти»
3.Блок ошибок: если логин занят, email некорректен или пароли не совпадают (красный текст)
Логика: при успешной регистрации редирект в /profile, данные отправляются POST /api/register

Основные блоки (для входа):
1.Форма: поля «Email», «Пароль», кнопка «Войти»
2.Ссылка «Забыли пароль?»
3.Ссылка «Нет аккаунта? Зарегистрироваться»
4.Блок ошибок: неверный email или пароль
Логика: при успешном входе редирект в /profile, получение JWT-токена

---

### Страница: Каталог товаров (/catalog)
Тип доступа: публичный
Основные блоки:
1.Шапка (как на главной)
2.Боковая панель фильтров:
• Тип устройства (клавиатуры, мыши, наушники) — чекбоксы
• Бренд (Logitech, Razer, HyperX и др.) — чекбоксы
• Цена (ползунок: от и до)
• Характеристики (для клавиатур: тип переключателей, подсветка; для 
мышей: DPI, тип сенсора) — динамически меняются в зависимости от выбранного типа
3.Сортировка: «По популярности», «По цене (возр.)», «По цене (убыв.)», «По рейтингу»
4.Сетка карточек товаров (12 на страницу): фото, название, цена, рейтинг, краткие характеристики, кнопка «В корзину»
5.Пагинация: «1 2 3 4 5 ...»
6.Информация о количестве найденных товаров: «Найдено 124 товара»
Динамический контент: карточки загружаются из GET /api/products?type=keyboard&brand=logitech&minPrice=1000&maxPrice=15000&sort=price_asc&page=1

Три состояния страницы:
1.Загрузка: скелетоны вместо карточек, фильтры заблокированы
2.С данными: отображаются реальные карточки товаров
3.Ошибка: сообщение «Не удалось загрузить товары» и кнопка «Попробовать снова»

---

Страница: Детальная страница товара (/product/:id)
Тип доступа: публичный
Основные блоки:
1.Галерея фото товара (основное фото + миниатюры)
2.Информация о товаре: название, бренд, рейтинг (звезды и количество отзывов), цена, наличие на складе
3.Вкладки: «Описание», «Характеристики», «Отзывы»
• Описание: текст с подробным описанием
• Характеристики: таблица (тип переключателей, подсветка, интерфейс и т.д.)
• Отзывы: список отзывов (имя пользователя, дата, текст, оценка)
4.Кнопки: «В корзину», «В избранное», «Сравнить»
5.Блок «Похожие товары»: 4 карточки аналогичных устройств
Динамический контент: данные из GET /api/products/:id, отзывы из GET /api/products/:id/reviews, похожие товары из GET /api/products/:id/similar

---

### Страница: Корзина (/cart)
Тип доступа: частный (только после входа)
Основные блоки:
1.Заголовок: «Корзина»
2.Список товаров в корзине:
• Фото, название, цена за единицу, количество (плюс/минус), итоговая цена за позицию, кнопка «Удалить»
3.Итоговая информация: общее количество товаров, стоимость без скидки, стоимость доставки, итоговая сумма
4.Промокод: поле ввода и кнопка «Применить»
5.Кнопка «Оформить заказ» (ведет на /checkout)
6.Пустое состояние: если корзина пуста — сообщение «В корзине пока ничего нет» и кнопка «Перейти в каталог»
Динамический контент: данные из GET /api/cart, обновление количества через PUT /api/cart/items/:id

---

### Страница: Оформление заказа (/checkout)
Тип доступа: частный (только после входа)
Основные блоки:
1.Многошаговая форма:
Шаг 1: Контактные данные (email, телефон)
Шаг 2: Адрес доставки (город, улица, дом, квартира, индекс)
Шаг 3: Способ оплаты (карта онлайн, наличные при получении)
Шаг 4: Подтверждение заказа (сводка по товарам, адресу и оплате)
2.Прогресс-бар: отображение текущего шага
3.Кнопки: «Назад», «Далее», «Подтвердить заказ» (на последнем шаге)
Логика: после подтверждения — отправка POST /api/orders, редирект на страницу заказа с номером заказа

---

### Страница: Личный кабинет (/profile)
Тип доступа: частный (только после входа)
Основные блоки:
1.Приветствие: «Здравствуйте, [Имя]!»
2.Навигация по профилю:
• Мои данные: email, телефон, имя (возможность редактирования)
• Мои адреса: список сохраненных адресов доставки
• История заказов: таблица с номером заказа, датой, статусом, суммой, кнопка «Подробнее»
3.Кнопка «Выйти из аккаунта»
Динамический контент: данные пользователя из GET /api/users/me, история заказов из GET /api/orders


---

### Страница: Избранное (/favorites)
Тип доступа: частный (только после входа)
Основные блоки:
1.Заголовок: «Избранное»
2.Сетка карточек товаров (аналогично каталогу) с пометкой «В избранном» и кнопкой «Удалить»
3.Пустое состояние: если избранное пусто — сообщение «Добавляйте товары в избранное» и кнопка «Перейти в каталог»
Динамический контент: данные из GET /api/favorites, удаление через DELETE /api/favorites/:productId

---

### Страница: Сравнение товаров (/compare)
Тип доступа: публичный
Основные блоки:
1.Заголовок: «Сравнение товаров»
2.Таблица сравнения:
• Строки: характеристики (тип устройства, бренд, цена, и специфические характеристики)
• Столбцы: выбранные товары (до 4)
• Вверху каждого столбца — фото и название товара, кнопка «Удалить из сравнения»
3.Если выбрано менее 2 товаров: сообщение «Выберите минимум 2 товара для сравнения»
4.Кнопка «Добавить товары» — ведет в каталог с возможностью выбора
Динамический контент: данные из GET /api/compare?ids=1,2,3

---
Страница: Админ-панель (/admin)
Тип доступа: частный (только role='admin')
Основные блоки:
1.Боковое меню:
Дашборд: статистика (количество заказов, пользователей, товаров)
Товары: список товаров с возможностью добавления/редактирования/удаления
Заказы: список всех заказов, изменение статуса
Пользователи: список пользователей, изменение роли, блокировка
Категории: управление категориями товаров
2.Таблица товаров: ID, название, цена, наличие, кнопки «Редактировать»/«Удалить»
Форма добавления товара: название, описание, цена, категория, характеристики (JSON), загрузка фото
Графики: «Продажи по дням», «Популярные категории» (для дашборда)
Динамический контент: все данные через GET /api/admin/* с проверкой прав администратора

## 3.5. Схема данных и API

### Основные модели данных

#### Пользователь (User)
Хранит данные для аутентификации и управления доступом

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | INTEGER | Уникальный идентификатор (автоинкремент) |
| `login` | TEXT | Уникальный логин пользователя |
| `email` | TEXT | Уникальный email пользователя |
| `slug` | TEXT | URL-идентификатор |
| `password` | TEXT | Хэш пароля (не хранится в открытом виде) |
| `role` | TEXT | Роль: `'user'` или `'admin'` |
| `created_at` | DATETIME | Дата регистрации |

#### Товар (Product) основная сущность
Хранит информацию о бассейнах для каталога

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | INTEGER | Уникальный идентификатор товара |
| `name` | TEXT | Название товара |
| `description` | TEXT | Подробное описание |
| `price` | INTEGER | Цена |
| `brand` | TEXT | Бренд |
| `rating` | REAL | Средний рейтинг (0.0–5.0) |
| `created_at` | DATETIME | Дата добавления в каталог |

### Ключевые эндпоинты API

#### Аутентификация

| Метод | Эндпоинт | Описание | Тело запроса | Ответ |
|-------|----------|----------|--------------|---------------|
| `POST` | `/api/register` | Регистрация нового пользователя | `{ "login": "user", "password": "123456" }` | `{ "message": "Успех", "userId": 1 }` |
| `POST` | `/api/login` | Вход в аккаунт, выдача токена | `{ "login": "user", "password": "123456" }` | `{ "token": "jwt...", "user": {...} }` |




#### Товар (основная сущность)


| Метод | Эндпоинт | Описание | Параметры | Ответ |
|-------|----------|----------|-----------|---------------|
| `GET` |`/api/products` | Получить список товаров с фильтрацией | `?category=keyboard&brand=logitech&minPrice=1000&maxPrice=15000&sort=price_asc&page=1&limit=12` | { "products": [...], "total": 124, "page": 1, "totalPages": 11 }` |
| `GET` | `/api/products/:id` | Получить детали одного товара| `:id` — номер товара в URL | `{ "id": 1, "name": "Logitech G502", "price": 4500, ... }}` |
